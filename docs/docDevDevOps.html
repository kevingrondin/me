Gitlab CI/CD

Pre-requis

Documentation officiel

Supprimer les anciennes installation et config

# Supprimer
apt remove gitlab-runner

# Purger
apt purge gitlab-runner

Supprimer les depots gitlab_runner dans /etc/apt/sources.list.d/

Installation du runner

curl -L https://packages.gitlab.com/install/repositories/runner/gitlab-runner/script.deb.sh | sudo bash

le repertoire ./home/gitlab-runner doit être vide vérifier avec

ls -lah

Installation

apt-get install gitlab-runner

Supprimer le skeleton ( Peux éviter des problèmes )

export GITLAB_RUNNER_DISABLE_SKEL=true; sudo -E apt-get install gitlab-runner

Verifier qu'il n'y ait pas de service en cours restant

gitlab-runner list

Si des services sont présent :

gitlab-runner unregister --all-runners

# Ou pour forcer leur supression
gitlab-runner verify --delete

Inscription du runner

gitlab-runner register
# exemple : https://gitlab.mycompany.org/

On va vous demandez le token ou jeton d'inscription ainsi qu'un tag qui seras présent dans votre fichier yml à la racine de votre projet

Dans paramètre et Intégration et livraison continu et executeur pour avoir tout les informations

Ensuite on vous demande executor pour rester simple on va choisir shell

shell

Tester

Faire votre commit avec votre projet et verifier avec

journalctl -f -u gitlab-runner

Exemple code intégration

.gitlab-cy.yml

stages:
  - config
  - install

install_packages:
  stage: config
  tags:
    - myapp_tag
  script:
    - '[[ -f /home/gitlab-runner/myapp/myapp_archive.tar.gz ]] && rm /home/gitlab-runner/myapp/myapp_archive.tar.gz'
    - '[[ -f /home/gitlab-runner/myapp/myapp_archive.tar.gz ]] && tar -cvf myapp_archive.tar /home/gitlab-runner/myapp'
    - '[[ -f /home/gitlab-runner/myapp/myapp_archive.tar.gz ]] && gzip myapp_archive.tar'
    - rm -r /home/gitlab-runner/myapp
    - cd .. && mv myapp /home/gitlab-runner/
    
deploy_app:
  stage: install
  tags:
    - myapp_tag
  script:
    - cd /home/gitlab-runner/myapp/ && npm i
